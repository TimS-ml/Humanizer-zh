# 技术写作深层检查设计

## 背景

Humanizer-zh 当前主要检查句子级 AI 写作模式. 它能识别填充词, 直译隐喻和机械句式, 但无法稳定发现以下问题:

- 一篇文章同时展开多套框架, 却没有明确主线.
- 私人术语和隐喻形成封闭系统, 读者必须先学习作者自己的词汇.
- 公式, 定理式措辞或数学符号强于实际推导和可测量定义.
- 论文结果, 作者解释, 工作假说和待核来源使用同一种确定语气.

这些问题无法靠逐句润色解决. Skill 需要先检查篇章和论证, 再处理句子表达.

## 设计

为完整技术笔记, 文章和多章节文档增加一个独立的"技术写作深层检查"阶段. 短段落, 单句改写, 术语表和纯资料清单默认跳过篇章检查, 直接执行现有 24 种句子级模式. 保留现有编号, 避免把内容审阅混入句式分类.

深层检查包含四项. 每项都需要说明适用条件, 检查信号, 命中标准和处理动作:

1. 叙事主线: 先识别目标读者, 文档类型和核心问题, 再标记各章节承担定义, 机制, 证据, 推论或索引中的哪项职责. 与核心问题无关, 没有过渡, 重复已有内容, 或在定义前使用概念时命中. 审阅时指出断点; 改写时先调整顺序和层级, 再润色句子.
2. 私人术语系统: 标准英文术语保留原词. 只有缩写且目标读者可能不熟悉时才补全称和一句白话解释. 领域内通行的隐喻不自动命中. 作者生造标签若缺少白话定义, 没有对应机制, 继续派生一串新隐喻, 或直接充当论据, 才判为问题. 若标签已经先解释机制并能稳定指代概念, 可以保留.
3. 形式化强度: 公式检查变量, 作用域和假设是否定义; 定理式结论检查条件, 推导或来源; 经验性结论检查操作化定义与测量方法. 正交, 同构, 守恒, 偏导数等词用于严格数学含义时不改, 用于包装松散类比时才命中. Skill 检查表达与论证是否匹配, 不负责代替领域专家证明公式.
4. 证据层级: 区分来源报告的结果, 来源作者的解释, 笔记作者的综合推断, 工作假说, 预测和来源待补. 精确数字需要能追溯到引用, 原始记录或明确计算过程. 证据不足时降低措辞强度或标记 `[来源待补]`, 绝不补造来源. 若文末 Caveat 限制前文结论, 审阅时同时指出两处, 改写时把限制移到首次提出结论的位置.

## 行为变化

- Skill 先按输入长度和任务类型判断是否运行深层检查. 深层检查没有命中时, 仍继续扫描 24 种句子级模式.
- 用户要求审阅, 诊断或列问题时, 进入审阅模式. 只输出按严重性排序的问题, 每条包含位置, 原文或术语, 原因和修正方向, 不提供替换后的全文.
- 用户要求改写或润色时, 进入改写模式. 完整技术文档先修主线和概念顺序, 再处理句子级模式, 最后输出改写文本和可选摘要.
- 用户没有明确指定模式时, 沿用现有默认行为, 进入改写模式.
- 重组主线若需要替作者选择立场, 删除有争议内容或改变技术结论, 先提问, 不擅自决定.
- 私人隐喻不能直接承担论证. 应先陈述实际机制, 再决定是否保留助记标签.
